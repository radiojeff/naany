Notes:
The required fields for this tool per ADIF file are listed below.
  • EOH - The header of the ADIF is separated from the records. The software doesn't technically need this record, but it is by default part of all ADIF files, so the software relies on finding this token before evaluating any QSO record.
  • QSO_DATE - Required, obviously to check the date of each QSO record against the specified year you entered for the NAANY award calculation.
  • DXCC - Required. The logging software must insert a field for the numeric identity of the DXCC entity. Most logging programs do this. Without the DXCC numeric valie per record, the software cannot feasibly reverse lookup the CALL to determnine the continent. It could use the CALL field, but that would make the tool extremely slow and unreliable since not all call signs are in databases. Check your logging/conversion to ensure DXCC is present in each record.
  • CALL - Required to figure out if the contact has the properties for NAANY entries.
  • BAND - Required. Technically the band has no influence on the NAANY calculation, but this software requires it.
  • EOR - End of Record field. Required. All ADIF records must have it. All Records are terminated with this field. There is one EOR field per QSO record.
